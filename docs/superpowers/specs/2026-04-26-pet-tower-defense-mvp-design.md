# 宠物猫狗塔防游戏 MVP 设计文档

**日期：** 2026-04-26
**版本：** 1.0
**状态：** 已批准

---

## 一、项目概述

### 项目目标
开发一个极致最小化的 2D 塔防游戏 MVP，验证核心玩法可行性。

### 核心循环
玩家点击地图放置防御塔 → 敌人沿固定路线移动 → 塔自动攻击敌人 → 敌人到达终点失败 / 全部消灭则胜利

---

## 二、技术架构

### 引擎
- **Cocos Creator 2D**（渲染模式 2D）
- TypeScript 脚本

### 场景
- 单场景 `Game.fire`
- 场景包含：地图、防御塔、敌人、子弹、UI 文字

### 主循环
- 基于 `update(dt)` 的帧循环
- 敌人沿预设路径点移动
- 塔定时检测范围内敌人，发射子弹
- 子弹碰撞敌人造成伤害

### 碰撞检测
- 圆形半径检测（子弹与敌人）

## 四、数据结构

### 防御塔 (Tower)
```
{
  x: number,           // 位置
  y: number,
  range: number,        // 攻击范围 150px
  damage: number,       // 伤害 10
  fireRate: number,     // 射速 1秒
  lastFireTime: number  // 上次射击时间
}
```

### 敌人 (Enemy)
```
{
  x: number,
  y: number,
  hp: number,           // 血量 100
  maxHp: number,
  speed: number,        // 移动速度 50px/s
  pathIndex: number,    // 当前路径点索引
  hpBar: cc.Node        // 血条
}
```

### 波次 (Wave)
```
{
  totalEnemies: number, // 总数 10
  spawned: number,      // 已生成
  spawnTimer: number,   // 生成间隔 1s
  enemies: Enemy[]      // 存活敌人列表
}
```

### 游戏状态 (GameState)
```
{
  running: boolean,
  won: boolean | null,
  placedTower: Tower | null,
  wave: Wave
}
```

## 五、路径设计

固定单一路线路径，从屏幕左侧进入，右侧出去：

```
(0, 270) → (200, 270) → (200, 100) → (600, 100) → (600, 270) → (960, 270)
```

## 六、游戏流程

1. **初始化** — 加载场景，初始化 GameState
2. **等待放置** — 显示地图，玩家点击放置猫塔
3. **波次开始** — 点击"开始"按钮，启动波次
4. **敌人生成** — 每秒生成一个敌人，共 10 个
5. **敌人移动** — 沿路径点移动
6. **塔攻击** — 每秒对范围内敌人发射子弹
7. **子弹命中** — 子弹到达敌人位置，造成伤害
8. **死亡处理** — 敌人血量归零，移除并播放死亡动画
9. **结束判断** — 10 个敌人生成且全部死亡 = 胜利；任意敌人到达终点 = 失败
10. **结果显示** — 显示 Victory! 或 Defeat!

## 八、验收标准

- [ ] 玩家可以点击地图放置猫塔（仅限一次）
- [ ] 点击"开始"按钮后，敌人从左侧进入并沿路径移动
- [ ] 猫塔自动攻击范围内敌人
- [ ] 子弹命中敌人造成伤害，敌人血条减少
- [ ] 敌人死亡后移除
- [ ] 敌人到达终点显示 "Defeat!"
- [ ] 所有敌人生成且死亡后显示 "Victory!"
- [ ] 微信小游戏可运行（主包 < 4MB）
